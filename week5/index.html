Week 5 Notes

  • ‘use strict’; can be placed at the top of a block of code and run the code in a stricter mode. This helps bring out more possible errors and can be used inside function blocks to only use strict mode in that function.
  • "debugger;" creates a breakpoint that can be used to step through browser JavaScript code execution.
  • You can create and throw your own exceptions to ensure your code executes correctly. These errors should then be handled within try-catch(-finally) statements.
  • Many developers choose to program using a Test-Driven mindset. Steps for this are as follows:
  1. Write tests (that initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test the refactored code
  5. Write more tests for new features
